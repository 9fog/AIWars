<b>*** Как запускать бота на тест ***</b>
1. Скачиваете себе песочницу. 
2. Создаете рядом с Sandbox новый пэкедж, например "MyMegaBot"
3. В этом пэекедже создаете класс MyBot.java, расширяющий класс Sandbox.AbstractBot
4. Делаете копию этого пэкеджа, назвав его "MyMegaBot2" - это будет ваш противник. В качестве
противника можете так же использовать одного из ботов, идущих вместе с Sandbox - MyBot1
или MyBot2.
5. В Config.java песочницы с массиве BOTS пишете: 
BOTS = { "MyMegaBot", "MyMegaBot2" } // бой будет между двумя вашими ботами
 или
BOTS = { "MyMegaBot", "MyBot1" } //Бой будет между вашим ботом, и дефолтным из песочницы
6. Убедитесь, что адрес сервера указан 46.4.189.241, в порт 21001. Блок "//Online settings"
может быть закомментирован. Если да - раскомментируйте, а "//Local setting" можете удалить.
7. Запускаете класс Starter из песочницы - это главный класс.
8. Если ругается, что не хватает класса json-smart.... - укажите этот jar в classpath при запуске.
Сам jar прилагается.

9. Итак, запустились, бой начался, продлился какое-то время, и завершился. В последних
строках лога в консоли вы увидите URL-адрес. По этому адресу вы можете просмотреть
визуализацию боя.
10. Если все ок - самое время начать писать своего бота. Для начала - можете
посмотреть, как устроен MyBot1 из песочницы.
11. Чтобы закачать своего бота на сервер для проведения онлайн боев - надо подождать, пока
мы реализуем эту фичу. :)

12. Удачи и фана! :)


<b>*** package Sandbox ***</b>
Там лежит сама песочница. Ничего менять там не стОит, кроме файла Config.java

Sandbox.AbstractBot - прототип главного класса для ваших ботов.
Sandbox.Config - здесь указывается, между какими ботами идет бой. На какой сервер соединяться, и т.п.
Sandbox.Connector - Класс реализует сетевое взаимодействие с игровым сервером
Sandbox.Event* - вэ-ошки событий, происходящих в бою. Их параметры нужно анализировать боту, 
чтобы принимать верные решения
Sandbox.Object* - объекты карты
Sandbox.Simulator - класс, реализующий общение между вашими ботами и классом Connector. Избавляет вас 
от необходимости вникать в протокол обмена с сервером.
Sandbox.Starter - запускаемый класс песочницы.


<b>*** package MyBot ***</b>
В этом пэкедже лежит все, что касается логики вашего бота. Обязательно должен 
присутствовать класс MyBot.java, расширяющий абстрактный класс Sandbox.AbstractBot из песочницы.
